Das Hookesche Gesetzt

Die Spannung σ beschreibt das Verhältnis einer Kraft F zu einer Fläche A.

σ = F/A

Die einwirkende Kraft bewirkt eine Formänderung. Im Falle eines eindimensionalen Stabes kann diese Formänderung durch eine Länge Δl beschrieben werden. Das Verhältnis zur ursprünglichen Länge des Stabes l0 ist die Dehnung ε.

ε = Δl / l0

Der lineare Zusammenhang zwischen der Spannung σ und der Dehnung ε wird durch das Elastizitätsmodul E beschrieben.

σ = E ∙ ε

Alle Kräfte im Gleichgewicht

Ist die Summe aller auf einen Körper einwirkenden Kräfte gleich Null, so befindet sich nach dem ersten newtonschen Gesetzt (siehe grauer Kasten rechts) dieser Körper in Ruhelage. Dieser spezielle Zustand des Gleichgewichts wird in der Statik einem Teilgebiet der Mechanik untersucht.

Ein einfaches Rechenbeispiel soll das Prinzip der Statik illustrieren. Drei Punkte A, B und C sind durch die starren Stäbe I (von A nach B), II (von B nach C) und III (von C nach A) zu einem Fachwerk verbunden. Der Punkt A ist durch ein Festlager mit der Umgebung verbunden. Im Punkt B wirkt eine Kraft F im Winkel α. Der Punkt C wird durch ein Loslager festgehalten und kann sich in der X-Achse frei bewegen.

Alle drei Punkte sind in Ruhelage, also muss die Summe aller Kräfte in den jeweiligen Punkten Null betragen. Dies kann für jeden Punkt und jede Richtung als Gleichung formuliert werden. Dabei werden die Kräfte in den Stäben als FI, FII und FIII bezeichnet. Die dazugehörigen Winkel σ und ρ ergeben sich aus der Konstruktion.

Der Kraft F im Punkt B wirken die zwei Stabkräfte FI und FII entgegen.

∑ Fx = 0 = F ∙ cos(α) + FI ∙ cos(σ) + FII ∙ cos(ρ)
∑ Fy = 0 = F ∙ sin(α) + FI ∙ sin(σ) + FII ∙ sin(ρ)

Im Punkt A befindet sich das Festlager. Die Kräfte im Lager werden als FAx und FAy bezeichnet. Der Stab III liegt horizontal, also wirken nur Kräfte in X-Richtung.

∑ Fx = 0 = FAx + FI ∙ cos(σ) − FIII
∑ Fy = 0 = FAy + FI ∙ sin(σ)

Zum Schluss noch Punkt C. Dieser ist durch ein Loslager befestigt, also kann nur eine Kraft FCy in Y-Richtung aufgenommen werden.

∑ Fx = 0 = FII ∙ cos(ρ) − FIII
∑ Fy = 0 = FII ∙ sin(ρ) + FCy

Gegeben sind die Größen F, α, σ und ρ. Durch die oben aufgestellten sechs Gleichungen, können nun die sechs Unbekannten FAx, FAy, FCy, FI, FII und FIII gefunden werden. Wie dies elegant und schematisch mithilfe mathematischer Verfahren gelöst werden kann, wird im nächsten Abschnitt beschrieben.

 
 

Die drei newtonschen Gesetze

  • Wirkt keine Kraft auf einen Körper so verharrt dieser entweder in Ruhe oder bewegt sich mit gleichbleibender Geschwindigkeit geradlinig weiter.
  • Die Kraft F ist das Produkt aus der Masse m und der Beschleunigung a: F = m ∙ a.
  • Kräfte treten immer paarweise auf. Übt ein Körper A auf einen anderen Körper B eine Kraft FA→B aus (actio), so wirkt eine gleich große, aber entgegen gerichtete KraftFB→A von Körper B auf Körper A (reactio) FA→B = −FB→A.

Kräfteparallelogramm

Die Wirkung einer Kraft kann in zwei Teilkräfte mit gleichem Angriffspunkt unterteilt werden. So kann eine durch Betrag F und Winkel α definiert Kraft in eine X-Komponente Fx und eine Y-Komponente Fy geteilt werden.

Fx = F ∙ cos(α)
Fy = F ∙ sin(α)

Festlager und Loslager

Ein Körper ist durch Lager mit seiner Umgebung verbunden. Durch ein Lager wird also die Bewegungsfreihet des gelagerten Körpers eingeschränkt. Das Festlager sperrt die Bewegung (nicht aber die Rotation) in alle Richtungen und das Loslager lässt die Bewegung in eine Richtung (wie bei einer Schublade) weiterhin zu.

Lineare Gleichungs­systeme

Gleichungssysteme, in welchen die Unbekannten nur durch Addition und Subtraktion zusammenhängen, werden linear genannt und können in Matrixform umgeschrieben werden. So auch die sechs Gleichungen im Abschnitt zuvor.

cos(σ) cos(ρ) 0 0 0 0 ∙ FI = −F ∙ cos(α)
sin(σ) sin(ρ) 0 0 0 0 FII −F ∙ sin(α)
cos(σ) 0 −1 1 0 0 FIII 0
sin(σ) 0 0 0 1 0 FAx 0
0 cos(ρ) −1 0 0 0 FAy 0
0 sin(ρ) 0 0 0 1 FCy 0

Nun kann das Gleichungssystem mittels der nachfolgenden Methode gelöst werden.

Das gaußsche Eliminationsverfahren

Gleichungen können innerhalb des Systems miteinander addiert und mit einem Faktor multipliziert werden ohne dass dabei die Lösung verändert wird. Dadurch wird es möglich die Matrix in eine rechte obere (oder linke untere) Dreiecksform umzuwandeln, sofern die Anzahl der Spalten mit der Anzahl der Zeilen übereinstimmt. In dieser Form kann die Lösung direkt durch Rückwärtseinsetzen abgelesen werden.

Sei A eine n × n Matrix, x der gesuchte Vektor mit den n Unbekannten und b ein Vektor mit Konstanten. Damit ist das Gleichungsystem definiert durch

A ∙ x = b

Für eine rechte obere Dreiecksform müssen die Elemente a21 bis an1 der ersten Spalte auf Null gebracht werden. Um dies für a21 zu erreichen, wird die erste Zeile von der zweiten abgezogen und mit dem Faktor λ = − a21 / a11 multipliziert (ist a11 = 0 so müssen zusätzlich Spalten getauscht werden). Diese Permutation kann als Multiplikation mit einer Permutationsmatrix P1 beschrieben werden.

1 0 ... 0 ∙ a11 a12 ... a1n = a11 a12 ... a1n
λ 1 ⁝ a21 a22 ⁝ 0 a22 ⁝
⁝ ⁝ ⁝ ⁝ ⁝ ⁝
0 ... 0 1 an1 an2 ... ann an1 an2 ... ann

Damit die Lösung sich nicht verändert, muss die Permutation auch auf die rechte Seite der Gleichung, also auf den Vektor b angewendet werden.

⇒ P1 ∙ A ∙ x = P1 ∙ b

Dies wird solange wiederholt bis die besagte rechte obere Dreiecksform erreicht ist. Dabei können die Permutationsmatrizen durch Multiplikation zu einer zusammengefasst werden P = P1 ∙ P2 ∙ ... ∙ Pn(n-1)/2

P ∙ A ∙ x = a11 a12 ... a1n ∙ x1 = P ∙ b = b ~= b~1
0 a~22 ⁝ x2 b~ 2
⁝ ⁝ ⁝ ⁝
0 ... 0 a~nn xn b~ n

Die letzte Zeile (oder Gleichung) enthält nun nur noch eine einzige Unbekannte xn und kann durch einfaches Umformen direkt bestimmt werden.

a~nn ∙ xn = b~ n
⇔ xn = b~n / a~nn

Die zweitletzte Zeile enthält zwei Unbekannte, wobei xn ja gerade bestimmt wurde. Also kann nach xn-1 umgeformt und bestimmt werden.

a~n-1,n-1 ∙ xn-1 + a~n-1,n ∙ xn = b~n-1
⇔ xn-1 = (b~n-1 − a~n-1,n ∙ xn) / a~n-1,n-1

Alle weiteren Unbekannten folgen analog. Wie dies automatisiert mittels Computern geschehen kann, wird im nächsten Abschnitt erklärt.

Multiplikation

Matrizen können untereinander multipliziert werden. Sei A eine m × n Matrix und B eine n × l Matrix. Die Multiplikation ist nur gültig, wenn die Spaltenanzahl n von A mit der Zeilenanzahl von B übereinstimmt. Dann gilt für die Elemente cij der resultierenden m × l Matrix C.

cij = ∑k=1 n aik ∙ bkj = ai1 ∙ b1j + ai2 ∙ b2j + ... + ain ∙ bnj

Vorsicht, die Matrixmultiplikation ist im Allgemeinen nicht kommutativ! Die Reihenfolge darf nicht getauscht werden.

A ∙ B ≠ B ∙ A

Implementation

Die Elemente einer Matrix werden in einem zweidimensionalen Array gespeichert. Der erste Index adressiert die Zeile, der zweite die Spalte. Als elementare Operation wird als erstes die Matrixmultiplikation als Funktion umgesetzt.

function matrix_multiplication(A,B)
m = number of rows in A
n = number of columns in A
if n ≠ number of row in B
return
end
l = number of columns in B
C = zero m × l Array
for i = 1 to m
for j = 1 to l
for k = 1 to n
C[i][j] = C[i][j]+A[i][k]∙B[k][j]
end
end
end
return C
end

Nun folgt das eigentliche Gauß-Verfahren. Dabei enthält die Variable A die Matrix A und die Variable b den Vektor b des Gleichungssystems A ∙ x = b. Der Algorithmus erzeugt spaltenweise Permutations­matrizen, welche fortlaufend mit der Matrix A und dem Vektor b multipliziert werden.

n = number of rows or columns in A
for i = 1 to n
P = identity matrix of dimension n
for j = i+1 to n
P[i][j] = −A[j][i]/A[i][i]
end
A = matrix_multiplication(P,A)
b = matrix_multiplication(P,b)
end

Zum Schluss folgt das Rückwärtseinsetzen.

x = zero vector of dimension n
for i = n backwards to 1
x[i] = b[i]
for j = i+1 to n
x[i] = x[i]−A[i][j]*x[j]
end
x[i] = x[i]/A[i][i]
end

Die Lösung des Gleichungssystem A ∙ x = b ist nun in der Variable x gespeichert.

Die hier vorgestellte Implementation enthält viel Optimierungspotential und würde in einem realen Programm mit den unterschiedlichsten Verbesserungen versehen werden. Wenn du wissen möchtest wie lineare Gleichungssysteme effizienter gelöst werden können, bist du im CSE Studium genau richtig.

 

Kleines Pseudocode-Programmierer-Glossar

Ein Programmcode wird oft als Pseudocode dokumentiert. Die dabei verwendete Syntax entspricht nicht einer spezifischen Programmiersprache, daher kann der Code so nicht direkt von einer Maschine interpretiert werden. Er dient zur Beschreibung der Programmstruktur unabhängig der verwendeten Technologie und kann als Grundlage zur Implementation in jeder beliebigen Programmiersprache verwendet werden.

Array Eine einfache Datenstruktur, in welcher Daten als eine Liste abgebildet werden. Jeder Listeneintrag enthält einen eindeutigen Index.
if Dadurch kann im Code mittels einer Bedingung eine Verzweigung erzeugt werden. Ein Teil des Codes wird ausgeführt falls die Bedingung wahr ist, der else-Teil wird ausgeführt falls die Bedingung nicht zutrifft.
for Die for-Schleife wird genutzt um Code mehrfach auszuführen. Dabei wird ein Index kontinuierlich hochgezählt.
while Programmcode innerhalb der while-Schleife wird solange die Bedingung wahr ist ausgeführt.
function Gleicher Code der mehrfach benutzt wird, kann in einer Funktion ausgelagert werden. Eine Funktion hat mehrere Eingangsparameter und meist nur einen Rückgabewert.
